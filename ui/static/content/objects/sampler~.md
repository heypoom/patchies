Audio sampler with recording and playback. Records audio from connected sources
into a buffer and plays it back with loop points, playback rate, and detune
control.

![Patchies chromatic sampler demo](/content/images/chromatic-sampler.webp)

> Try this patch [in the app](/?id=ifeac3e32g9vfm1)!
> A sampler that changes playback speed based on MIDI notes.

## Controls

- **Record** (circle): Start recording
- **Play**: Play the sample
- **Settings** (gear): Configure start/end, gain, rate, detune, loop, and note-off behavior

## Loading Samples

Drop an audio file into the sampler to load it.

## Playback Messages

Send `{ type: "bang" }` to play immediately. For sample-accurate triggering
from sequencers and clocks, include `time`:

```js
{ type: "bang", time: audioContextTime }
```

`bang` also accepts optional playback fields:

```js
{ type: "bang", time: audioContextTime, offset: 0, duration: 0.25, value: 0.5 }
```

- `time` schedules playback at an absolute `AudioContext.currentTime` timestamp
- `offset` starts from a position in the sample buffer, in seconds
- `duration` plays a specific amount of source-buffer audio, in seconds
- `value` scales this playback voice's amplitude

All four fields are optional. When omitted, playback uses the sampler's current
start/end settings and normal gain.

Sequencer value output with **Audio lookahead** sends timed `bang` messages
with `value`, so it can drive sampler velocity directly:

```js
{ type: "bang", time: audioContextTime, value: 0.75 }
```

`set` messages are not playback triggers. Use `setGain` to set the sampler's
built-in output level. This scales all voices after per-trigger value and MIDI
velocity:

```js
{ type: "setGain", value: 0.75 }
```

You can also send a non-negative number to trigger playback with that gain
multiplier:

```js
0.5  // half amplitude
1.0  // normal amplitude
2.0  // twice the amplitude
```

`sampler~` also accepts MIDI-style note messages:

```js
{ type: "noteOn", note: 60, velocity: 127 }
{ type: "noteOff", note: 60 }
{ type: "setNoteOffMode", value: "held" }
```

- `note` maps to playback rate, with note `60` playing the sample at original
  pitch. Each semitone changes playback rate by `2 ** (1 / 12)`.
- `velocity` maps to gain as `velocity / 127`.
- `noteOff` is ignored by default for one-shot sampler behavior.
- Set Note Off mode to `held` when you want `noteOff` to stop active voices for
  the matching note.

## Float32Array Input

The sampler can receive audio samples directly as a `Float32Array` via its
message inlet. This is useful for loading samples generated by other nodes
like [uiua](/docs/objects/uiua).

**Format:**

- Values should be normalized audio samples in the range `-1.0` to `1.0`
- Samples are interpreted at the current audio context sample rate
  (typically 44100 or 48000 Hz)
- **Mono only** - if the source is stereo, only the first channel is used

**Example workflow:**

```text
[uiua: ×0.3 ○×τ×220 ÷⟜⇡ 44100]  →  [sampler~]  →  [dac~]
                                        ↑
                                  [button: play]
```

The uiua node generates a 1-second 220Hz sine wave, sends it as Float32Array
to the sampler, then you click "play" to hear it.
