Audio sampler with recording and playback. Records audio from connected sources
into a buffer and plays it back with loop points, playback rate, and detune
control.

![Patchies chromatic sampler demo](/content/images/chromatic-sampler.webp)

> Try this patch [in the app](/?id=ifeac3e32g9vfm1)!
> A sampler that changes playback speed based on MIDI notes.

## Controls

- **Record** (circle): Start recording
- **Play**: Play the sample
- **Settings** (gear): Configure start/end, loop, rate, detune

## Loading Samples

Drop an audio file into the sampler to load it.

## Float32Array Input

The sampler can receive audio samples directly as a `Float32Array` via its
message inlet. This is useful for loading samples generated by other nodes
like [uiua](/docs/objects/uiua).

**Format:**

- Values should be normalized audio samples in the range `-1.0` to `1.0`
- Samples are interpreted at the current audio context sample rate
  (typically 44100 or 48000 Hz)
- **Mono only** - if the source is stereo, only the first channel is used

**Example workflow:**

```text
[uiua: ×0.3 ○×τ×220 ÷⟜⇡ 44100]  →  [sampler~]  →  [dac~]
                                        ↑
                                  [button: play]
```

The uiua node generates a 1-second 220Hz sine wave, sends it as Float32Array
to the sampler, then you click "play" to hear it.
